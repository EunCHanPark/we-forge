# Harness Core Index

## Purpose
이 문서는 repository-level harness 문서의 진입점이다.
먼저 이 문서를 읽고, 필요한 하위 문서로 이동한다.

## Read Order
1. `constitution.md` — 최상위 원칙 + 문서 충돌 우선순위
2. `governance.md` — 문서 경계와 변경 규칙
3. `core-beliefs.md` — B1~B12 원칙
4. `git-conventions.md` — 커밋/브랜치 규칙
5. `repository-architecture.md` — 모노레포 구조, 의존성 방향, 배포 원칙
6. `repository-security.md` — 공통 보안 기준
7. `repository-reliability.md` — 공통 안정성/에러 처리 기준

## Use Rule
- 공통 규칙은 여기서 찾는다.
- 앱별 규칙은 `harness/docs/` 아래에서 찾는다.
- 문서가 충돌하면 `constitution.md`를 먼저 따른다.
- 역할 규칙은 `roles/`, 워크플로우는 `workflows/`를 읽는다.
