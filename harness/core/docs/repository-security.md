# Repository Security

## 1. 시크릿 관리
- 코드, 커밋 히스토리, 로그에 시크릿을 포함하지 않는다.
- 시크릿은 환경변수 또는 Secret 리소스로 주입한다.
- `.env` 파일은 `.gitignore`에 포함. `.env.example`에 키 목록만 커밋한다.
- 시크릿에 하드코딩 fallback을 두지 않는다. 없으면 기동 실패.

## 2. 인증·권한
- 모든 외부 노출 엔드포인트는 인증을 거친다. 공개 엔드포인트는 명시적으로 표시한다.
- 권한 체크 실패 시 최소 정보만 반환한다. 내부 스택 트레이스 노출 금지.
- 인증 실패, 권한 거부 등 보안 이벤트는 반드시 로그에 기록한다.

## 3. 신뢰 경계
- 외부에 노출되는 서비스와 엔드포인트는 trust-boundaries 문서에 명시한다.
- 문서에 없는 외부 노출은 허용하지 않는다.
- 앱별 신뢰 경계 세부사항은 app-level 문서에 명시한다.

## 4. 입력 검증
- 신뢰하지 않는 모든 외부 입력은 검증한다.
- SQL Injection: 파라미터 바인딩 사용. 문자열 결합 쿼리 금지.
- XSS: 프레임워크 기본 이스케이핑 유지. 직접 HTML 삽입 금지.

## 5. 데이터 보호
- 비밀번호는 해시 저장. 평문 금지.
- PII(개인식별정보)는 로그에서 마스킹한다.
- 통신은 HTTPS를 강제한다.
- API 응답은 필요한 필드만 반환한다.

## 6. 의존성 보안
- 새 의존성 추가 전 사용자 승인을 받는다 (Non-Negotiable 7)
- 주기적 취약점 스캔을 수행한다.
- 앱별 세부 보안 규칙은 app-level 문서에 명시한다.
